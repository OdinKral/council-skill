# Values Alignment Review Workflow

## Trigger

User says: "values review", "alignment check", "telos check", or "am I aligned?" — optionally followed by a topic or specific concern.

## Purpose

Check whether recent actions and decisions align with your stated values and goals. This is calibration, not judgment. The review uses three voices: the Critic (audit), the Sage (drift detection), and the Coach (corrective action).

## Execution Steps

### 1. Load Reference Values

Read your values document, charter, or mission statement — whatever defines what you're optimizing for.

### 2. Load Archetypes

Read:
- `archetypes/Critic.md` (bias/self-deception detection)
- `archetypes/Sage.md` (drift/meta-pattern analysis)
- `archetypes/Coach.md` (activation/momentum)

### 3. Gather Context

Ask the user (or infer from recent work):
- What decision, action, or pattern are you checking?
- What period are you reviewing? (day / week / month)
- Any specific concern triggering this review?

### 4. Empirical Grounding Check

Before archetype analysis, establish factual ground:
- **What evidence supports your current direction?**
- **What would falsify your assumptions?**
- **Are you reasoning from data or from anxiety?**
- **What is physically, measurably true right now?**

### 5. Present to Each Voice

**Voice 1: The Critic — Alignment Audit**
- Check actions against stated values
- Name specific biases or self-deceptions detected
- Identify where stated values and actual behavior diverge
- Surface rationalizations and comfortable blind spots

**Voice 2: The Sage — Drift & Depth Scan**
- Analyze patterns across recent behavior
- Is speed increasing while depth decreases?
- Are you serving the goal or serving the system?
- Is complexity increasing unnecessarily?

**Voice 3: The Coach — Corrective Action**
- Identify the smallest realignment action (not the biggest overhaul)
- Translate findings into concrete next steps
- Acknowledge what is working — alignment is not only about correction

### 6. Output Format

```markdown
# Values Alignment Review

**Date:** YYYY-MM-DD
**Period:** [day/week/month reviewed]
**Trigger:** [what prompted this review]

## Empirical Ground
[What is factually true vs. what is assumed]

## The Critic — Alignment Audit
[Biases named, value-behavior gaps identified]

## The Sage — Drift & Depth Scan
[Pattern analysis, drift indicators]

## The Coach — Corrective Action
[Smallest realignment action, what's working]

## Alignment Assessment

**Score:** [Aligned / Minor Drift / Significant Drift / Misaligned]

**Integrated Assessment:**
[Reconciled view from all three voices]

## 48-Hour Corrective Vector

1. [Concrete action 1]
2. [Concrete action 2]
3. [Concrete action 3]

## Session Value (fill within 48 hours)

- Action taken: [ ]
- Insight I wouldn't have reached alone: [ ]
- Right team for this question: [ ] (who was missing / who wasn't needed?)
```

### 7. Auto-Save

Save alongside other session outputs for the Sage to review during Meta Reviews.

## Notes

- This is calibration, not judgment. The human decides. The farmer cultivates.
- If significant drift detected, recommend a Council Session on the underlying topic.
- The empirical grounding step is not optional — it prevents the review from becoming abstract rationalization.
